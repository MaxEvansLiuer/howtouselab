🐙 什么是 OpenClaw？

OpenClaw 是一个开源的 AI Agent 框架，可以本地部署，支持多种大语言模型连接。可通过 Docker 快速部署，支持飞书、钉钉等 IM 集成。

环境要求：Docker Desktop (Windows/Mac) 或 Docker (Linux)，建议 4核8G 以上内存

🚀 Docker 部署教程

1

安装 Docker Desktop

Windows 用户：下载并安装 Docker Desktop

安装完成后启动 Docker，确认图标显示运行中

2

创建配置文件

新建一个文件夹如 openclaw，在其中创建 config.yaml 文件：

server:
  host: 0.0.0.0
  port: 8080

# 大模型配置 (选择一种)
llm:
  provider: openai  # 或 anthropic, azure, ollama 等
  api_key: your-api-key-here
  base_url: https://api.openai.com/v1  # 可选
  model: gpt-4o-mini

# IM 集成配置 (可选)
im:
  type: feishu  # 或 dingtalk, telegram
  app_id: your-app-id
  app_secret: your-app-secret
  verification_token: your-token
3

运行 Docker 容器

在文件夹中打开终端/命令行，运行：

docker run -d \
  --name openclaw \
  -p 8080:8080 \
  -v $(pwd)/config.yaml:/app/config.yaml \
  -v openclaw_data:/app/data \
  justlovemaki/openclaw:latest
4

验证安装

浏览器打开 http://localhost:8080

看到登录页面即表示成功！

🎉 部署完成！ 默认管理员账号 admin/admin123

⚙️ 常见配置说明

支持的 LLM 提供商

  • OpenAI - GPT-4o, GPT-4, GPT-3.5
  • Anthropic - Claude 3.5 Sonnet
  • Azure OpenAI
  • Ollama - 本地开源模型
  • 通义千问 - 阿里云
  • DeepSeek

飞书集成配置

1. 前往 飞书开放平台
2. 创建应用 → 开通权限: im:chat:message:send_as_bot
3. 发布应用 → 添加可用用户
4. 将 app_id 和 app_secret 填入配置

钉钉集成配置

1. 前往 钉钉开放平台
2. 创建企业内部应用 → 开通消息权限
3. 配置机器人 Webhook
4. 将相关凭证填入配置

🦞 什么是有道龙虾？

有道龙虾 (LobsterAI) 是网易有道推出的 AI Agent 产品，开箱即用，无需复杂配置。支持多种大模型接入，可快速创建个人 AI 助手。

特点：图形化界面友好，一键部署，无需命令行经验

🚀 Windows 一键安装

1

下载安装包

访问有道龙虾官网或各大技术博客获取下载链接

下载 Windows 安装包 (.exe)

2

运行安装程序

双击安装包，按提示完成安装

建议安装到 D:\LobsterAI 目录

3

启动服务

安装完成后会自动启动服务

或在开始菜单找到"LobsterAI"点击启动

4

访问管理界面

浏览器打开 http://localhost:8765

首次访问设置管理员账号

🎉 安装完成！ 接下来配置 AI 模型

🤖 配置大模型

在界面中配置

1

进入设置页面

点击左下角"设置"图标

2

选择模型提供商

支持：OpenAI、Anthropic、阿里通义、DeepSeek、Moonshot 等

3

填写 API Key

粘贴对应的 API Key

可选择设置默认模型

支持的模型

  • GPT-4o / GPT-4 - OpenAI
  • Claude 3.5 - Anthropic
  • 通义千问 Qwen - 阿里云
  • DeepSeek Chat - DeepSeek
  • Moonshot (月之暗面)

📱 集成飞书/钉钉

飞书集成

1. 设置 → 集成 → 飞书
2. 填写应用 App ID 和 App Secret
3. 开启消息接收
4. 在飞书中搜索并添加机器人为好友

钉钉集成

1. 设置 → 集成 → 钉钉
2. 填写钉钉机器人的 AgentId 和 Secret
3. 开启接收消息
4. 在钉钉中添加机器人

💡 常见问题

Q: 启动后无法访问？

A: 检查防火墙设置，确保 8765 端口未被占用

Q: API 调用失败？

A: 确认 API Key 正确，网络能访问对应服务商

Q: 如何更新版本？

A: 下载新版本覆盖安装，或在设置中检查更新
